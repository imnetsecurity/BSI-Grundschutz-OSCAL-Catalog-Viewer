Grundschutz++ OSCAL Viewer

Einführung, Erklärung aller Komponenten und Nutzung — alles in einer Seite.

Erste Schritte (in 30 Sekunden)

  1. Katalog laden: Oben URL eingeben (oder bestehende lassen) und Laden klicken – oder JSON hochladen.
  2. Control wählen: Links im Baum eine Zeile anklicken – rechts erscheint die Detailansicht. Mit ↑ / ↓ vorherige/nächste Control anspringen.
  3. Suchen: Im Query-Feld z. B. id:GC.2.* oder sec:hoch eingeben – die Liste filtert sofort.
  4. Export: „Selection mode“ an, Controls ankreuzen, Tab Selected → Export Profile oder Mini-Catalog.

Hauptfunktionen des Viewers

Der Viewer bietet folgende Kernfunktionen. Jede erfüllt einen klaren Zweck und ist an einer festen Stelle in der Oberfläche umgesetzt.

1. Katalog laden & anzeigen

Zweck: OSCAL-Catalog-JSON (z. B. Grundschutz++) ins Tool holen und als Baum + Metadaten anzeigen.

Wo: Header – URL-Feld, Button Laden, Button JSON hochladen. Nach dem Laden: linke Sidebar = Baum (gruppiert/sortierbar), Meta-Pills (Dokument, Version, Visible/Selected/Broken).

So geht’s: URL eintragen (GitHub-blob wird automatisch zu raw) → Laden klicken. Alternativ „JSON hochladen“ und lokale .json wählen. Beim Laden erscheint ein Overlay; danach zeigt der Baum alle Controls.

2. Suchen & Filtern

Zweck: Die angezeigte Liste auf bestimmte Controls einschränken (ID, Sicherheitsstufe, Aufwand, Klasse, Text).

Wo: Linke Sidebar – Query-Feld (einzeilig), Dropdowns sec_level, effort_level, class, Keyword. Optional Group by und Sort by.

So geht’s: Query z. B. id:GC.2.* sec:hoch oder has:broken eingeben (AND-Logik). Filter-Dropdowns und Keyword verfeinern. Reset setzt alles zurück. Die Zahl „X / Y“ zeigt sichtbare vs. gesamt.

3. Navigieren & Detailansicht

Zweck: Einzelne Control lesen (Statement, Guidance, Links, Params, Graph, Notizen).

Wo: Links Baum (Klick auf Zeile), rechts Tab Detail. Tastatur ↑ / ↓ für vorherige/nächste Control in der gefilterten Liste.

So geht’s: Im Baum eine Zeile anklicken → rechts erscheinen Statement, Guidance, Relationships, Params, Similarity (TF-IDF), lokaler Graph (Columns/Force, Zoom/Pan), Notes/Review/Evidence, Raw JSON. Deep-Link z. B. #id=GC.1.1 springt direkt zu dieser Control.

4. Auswahl (Selection) & Export

Zweck: Eine Teilmenge von Controls auswählen und als OSCAL-Profil, Mini-Katalog oder CSV exportieren.

Wo: Sidebar „Selection mode“ aktivieren → Checkboxen im Baum; oder im Detail „Select“. Tab Selected für Liste und Export-Buttons.

So geht’s: Selection mode an → Controls im Baum ankreuzen und/oder im Detail „Select“ klicken. Im Tab Selected: Export Profile (OSCAL-Profil-Skeleton), Export Mini-Catalog (JSON), Copy IDs, Export CSV. „Add visible“ fügt alle aktuell sichtbaren hinzu; „Prune“ entfernt nicht mehr vorhandene IDs.

5. Dashboard & Analysen

Zweck: Statistische Auswertungen und Visualisierungen (Verteilungen, Hubs, Broken Links, Ähnlichkeiten).

Wo: Tab Dashboard. Scope: All / Visible / Selected. Refresh, dup-thr, Button Near-duplicates.

So geht’s: Tab Dashboard öffnen. Verteilungen (sec_level, effort_level, class, groups), Quantile incoming/outgoing, Top Hubs, Scatter (incoming vs. outgoing), Tabelle Broken links. Near-duplicates: Schwellwert einstellen, Button klicken – TF-IDF-Worker berechnet ähnliche Paare.

6. Mapping & Traceability

Zweck: Beziehungen zwischen Controls als Matrix darstellen und Pfade zwischen zwei Controls finden.

Wo: Tab Mapping. Matrix (Scope, rel-Filter), Trace: From / To / Tiefe / rel, Button „Find paths“.

So geht’s: Matrix zeigt Zeilen = Quelle, Spalten = Ziel; Klick auf Zelle springt zur Relation. Trace: From- und To-Control wählen, Tiefe und optional rel setzen → „Find paths“ zeigt gefundene Pfade (bis 25).

7. Lint & Qualitätsprüfung

Zweck: Katalog auf Fehler und Warnungen prüfen (IDs, Titel, Links, Schema).

Wo: Tab Lint. Filter All/Errors/Warnings/Info, Tabelle mit Meldungen.

So geht’s: Tab Lint öffnen; Liste zeigt z. B. doppelte IDs, fehlende Titel, defekte Links, Schema-Abweichungen. Klick auf eine Zeile springt zur betroffenen Control. Refresh für erneuten Lauf.

8. Diff (Katalogvergleich)

Zweck: Zwei Katalogversionen vergleichen – neu, entfernt, geändert.

Wo: Tab Diff. A = aktuell geladen; B = per URL (Load B) oder Upload. Filter New/Removed/Changed, Suchfeld.

So geht’s: Katalog A ist bereits geladen. B per URL oder Datei-Upload laden. Tabelle zeigt New (nur in B), Removed (nur in A), Changed (in beiden, Inhalt geändert). Klick auf Zeile öffnet Detail der Änderungen.

9. Resources (back-matter)

Zweck: Ressourcen aus back-matter.resources durchsuchen und referenzierte/unreferenzierte anzeigen.

Wo: Tab Resources. Suche, Anzeige All/Referenced/Unreferenced, Tabelle, Detail bei Klick.

So geht’s: Tab Resources öffnen. Liste mit UUID, Titel, Referenzanzahl. Klick auf Zeile → Detail (Titel, Links, Citation, Referenced by). Export CSV. Deep-Link #resource=<uuid> springt zur Resource.

10. Review

Zweck: Review-Status, Reviewer, Tags und Notizen pro Control erfassen und als CSV/Markdown exportieren.

Wo: Im Detail-Panel: Bereich „Notes / Review / Evidence“. Tab Review für Übersicht und Export.

So geht’s: Control im Detail öffnen → Review state, Reviewer, Reviewed at, Status, Tags, Decision, Notes, Evidence eintragen → Save. Tab Review zeigt alle Einträge, Filter (z. B. Unreviewed), Suche. Export CSV und Export report (MD) für Dokumentation.

11. Workspace & Persistenz

Zweck: Zustand (URL, Tab, Filter, Query, Selection, Notizen) im Browser speichern; optional Offline-Cache.

Wo: Tab Workspace. JSON-Preview, Export/Import/Clear Workspace. Cache: Policy, Tabelle, Refresh/Clear. Reviewer-Name.

So geht’s: Alles wird automatisch im localStorage gehalten. Im Tab Workspace: Export sichert den kompletten Workspace als JSON; Import lädt einen gespeicherten Zustand. Cache (IndexedDB): Policy „Prefer cache“ oder „Network first“; bei erneutem Laden wird ggf. aus dem Cache geladen. Clear entfernt alle Cache-Einträge.

Was ist das?

Der Grundschutz++ OSCAL Viewer ist eine Standalone-Webanwendung (eine einzige HTML-Datei), mit der Sie OSCAL-Kataloge – insbesondere den Grundschutz++-Katalog des BSI – laden, durchsuchen, filtern und analysieren können.

  • OSCAL (Open Security Controls Assessment Language) ist ein offener Standard (NIST) zur maschinenlesbaren Darstellung von Sicherheitskatalogen, Profilen und Bewertungen.
  • Grundschutz++ ist der IT-Grundschutz-Katalog des BSI in OSCAL-Form („Stand der Technik Bibliothek“). Er enthält Controls (Anforderungen) mit IDs (z. B. GC.1.1), Statements, Guidance, Metadaten und Verknüpfungen.
  • Der Viewer zeigt diesen Katalog als Baum/Navigation, mit Detailansicht, Query-Sprache, Dashboard, Mapping, Lint, Diff, Review und Workspace – alles im Browser, ohne Server.

Wozu braucht man das?

  • Verständnis: Schnell verstehen, welche Anforderungen (Controls) der Katalog enthält und wie sie zusammenhängen.
  • Suche & Filter: Gezielt nach ID, Sicherheitsstufe, Aufwand, Klasse oder Text suchen (Query Language + Filter).
  • Auswahl (Selection): Controls auswählen und als OSCAL-Profil oder Mini-Katalog exportieren – z. B. für ein organisationsspezifisches Profil.
  • Qualität: Lint-Checks für IDs, Links, Schema; Diff zwischen zwei Katalogversionen.
  • Review: Lokaler Review-Workflow (Status, Reviewer, Tags, Notizen, Evidence-Links) mit Export als CSV/Markdown.
  • Traceability: Beziehungen zwischen Controls (Matrix, Pfadsuche) und Broken-Links-Analyse.
  • Offline/Performance: Optionaler IndexedDB-Cache für schnelleres erneutes Laden und Offline-Nutzung.

Wie nutzt man es? (Kurz)

  1. Katalog laden: URL eingeben (z. B. Grundschutz++-JSON von GitHub) und auf Laden klicken – oder JSON hochladen.
  2. Navigieren: Links erscheint der Katalog als Baum (gruppiert/sortierbar). Ein Klick auf eine Zeile öffnet die Detailansicht rechts. Pfeil ↑/↓ wechselt zur vorherigen/nächsten Control.
  3. Filtern/Suchen: Query-Feld (z. B. id:GC.2.* sec:hoch), Filter sec_level/effort_level/class und Keyword nutzen.
  4. Selection: „Selection mode“ aktivieren, Controls per Checkbox auswählen; im Tab Selected exportieren (Profile, Mini-Catalog, CSV).
  5. Weitere Tabs: Dashboard, Mapping, Lint, Diff, Resources, Review, Workspace je nach Bedarf nutzen.

Typische Workflows

Eine Control nachschlagen Katalog laden → links Baum nutzen oder Query (z. B. id:GC.1.1) → Detail rechts lesen.
Alle „kritischen“ Controls exportieren Filter sec_level oder Query sec:hoch → Selection mode → „Add visible“ → Tab Selected → Export Mini-Catalog oder Profile.
Zwei Katalogversionen vergleichen Katalog A laden → Tab Diff → B per URL oder Upload laden → New/Removed/Changed prüfen.
Review-Status erfassen Controls im Detail durchgehen → Notes/Review ausfüllen → Save → Tab Review für Übersicht und Export CSV/MD.

Kurz-Glossar

Control
Eine Anforderung im Katalog (z. B. GC.1.1) mit ID, Titel, Statement, Guidance, Parametern und Links.
Statement
Der normativ formulierte Text der Anforderung („Was muss erfüllt werden?“).
Guidance
Hilfestellung zur Umsetzung der Anforderung (nicht normativ).
rel
Typ einer Beziehung zwischen Controls (z. B. related, references).
Catalog
OSCAL-Dokument mit Metadaten, Gruppen und Controls.
Profile
OSCAL-Profil: referenziert einen Catalog und wählt Controls aus (with-ids, with-controls).
Broken Link
Ein ausgehender Link (href), der auf keine gültige Control oder Resource im Katalog zeigt.

Häufige Fragen

Der Katalog lädt nicht – was tun?
Bei externen URLs: CORS kann blockieren. Nutzen Sie „JSON hochladen“ oder eine URL, die CORS erlaubt. Bei GitHub: der Viewer wandelt blob-URLs automatisch in raw um.
Es werden keine Treffer angezeigt.
Query oder Filter lockern (z. B. Query leeren, Reset), oder sec_level/effort_level/class auf „(alle)“ stellen.
Die Selection bleibt leer.
„Selection mode“ muss aktiviert sein (Toggle in der Sidebar). Dann können Sie im Baum per Checkbox auswählen oder in der Detailansicht „Select“ klicken.
Wo werden meine Notizen gespeichert?
Im Workspace (localStorage im Browser). Export/Import im Tab Workspace sichert oder überträgt den kompletten Zustand.
Tipp: Lade einen Katalog (URL oder Upload), dann links eine Control anklicken oder mit ↑/↓ durch die Liste gehen – rechts erscheint die Detailansicht. Über die Tabs oben wechseln Sie zu Dashboard, Selected, Mapping usw.
